Handshake procedure

Before the Host and the Controller can exchange any meaningful data, they have to carry out a handshake. This handshake is initiated via the CDJ by using the Link menu: [LINK] > CONTROL MODE (CDJ) > USB MIDI. This will cause the CDJ to start sending USB-MIDI packets and the first Packet of the handshake procedure:

USB-Devices are by definition unable to send data to the host on their own. The host instead as to constantly pull for changes. So make sure the Software running on the host is doing that before connecting initiating the connection on the CDJ.

The meaning of the values at offset 0x08 to 0x0b is unknown.

At this point the host has to respond with the following message:

The packet consists of an extended header and a TLV structure which contains another three TLV structures in sequence. Values seen so far for manufacturer are "PioneerDJ" and "NativeInstruments" as well as "CDJ2000NXS2", "rekordbox" and "TraktorPro2" for the product.

TLV Types

TLV Type Description Direction

0x01

Contains the Name of the manufacturer.[1][2]

Bidirectional

0x02

Contains the Name of the Product.[1][2]

Bidirectional

0x03, 0x04, 0x05

Format currently unknown, but it is expected that just replaying the values in the diagram should work.

Bidirectional

0x11

Handshake initialization

CDJ→Host only

0x12

Handshake initialization response

Host→CDJ only

0x13

CDJ Handshake data packet

CDJ→Host only

0x14

CDJ Handshake data packet response

Host→CDJ only

0x15

CDJ Handshake completion

CDJ→Host only
1. Refers to the Software or Hardware depending on the message direction.
